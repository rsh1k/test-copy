

# Security Analysis: HIGH and CRITICAL CVEs in dotCMS

| No. | CVE Name | CVE Type (OWASP) | Description | Status | Explanation of Status |
|-----|----------|------------------|-------------|--------|----------------------|
| 1 | CVE-2025-66516 | A06:2021 – Vulnerable and Outdated Components | Critical vulnerability in Apache Tika PDF parser module allowing arbitrary code execution through malicious PDF files | **True Positive** | dotCMS uses Apache Tika extensively for content indexing and file processing (found in `dotCMS/core/dotCMS/src/main/java/com/dotcms/content/elasticsearch/business/ESContentletIndexAPI.java` and content analysis). No input sanitization or sandboxing detected for PDF processing. **Exploitability: HIGH** - PDF uploads are common in CMS platforms. |
| 2 | CVE-2025-54988 | A06:2021 – Vulnerable and Outdated Components | Critical vulnerability in Apache Tika PDF parser enabling remote code execution via crafted PDF documents | **True Positive** | Same component as CVE-2025-66516. Tika parser is initialized in `com.dotmarketing.util.ConfigUtils` and used throughout content processing without version checks or mitigations. **Exploitability: HIGH** - Direct attack vector through file upload functionality. |
| 3 | CVE-2024-40094 | A03:2021 – Injection | Denial of Service vulnerability in GraphQL Java library through deeply nested queries causing stack overflow | **True Positive** | dotCMS implements GraphQL API in `dotCMS/core/dotCMS/src/main/java/com/dotcms/graphql/` with no query depth limiting visible in `DotGraphQLProvider.java` or `GraphqlAPI.java`. Default configuration lacks complexity analysis. **Exploitability: MEDIUM** - Requires authenticated access to GraphQL endpoint but can cause service disruption. |
| 4 | CVE-2024-47072 | A08:2021 – Software and Data Integrity Failures | Remote code execution in XStream through deserialization of untrusted XML data | **True Positive** | XStream is used in `dotCMS/core/dotCMS/src/main/java/com/dotmarketing/portlets/workflows/business/WorkflowFactoryImpl.java` and cache serialization. No allow-listing or security restrictions detected in implementation. **Exploitability: MEDIUM-HIGH** - Depends on attacker's ability to inject XML into serialization paths. |
| 5 | CVE-2025-48734 | A08:2021 – Software and Data Integrity Failures | Property manipulation vulnerability in Apache Commons BeanUtils allowing class loader manipulation | **True Positive** | Commons BeanUtils used extensively in `com.dotmarketing.util.UtilMethods` and form processing. No property filtering detected in `BeanFactory.java` or related utilities. **Exploitability: MEDIUM** - Requires ability to control bean property names. |
| 6 | CVE-2023-24998 | A01:2021 – Broken Access Control | File upload vulnerability in Apache Commons FileUpload allowing unlimited file size leading to DoS | **True Positive** | FileUpload is core to dotCMS upload functionality in `com.dotmarketing.servlets.ajax.AjaxFileUploadServlet` and `com.dotcms.rest.api.v1.temp.TempFileResource`. | 7 | CVE-2025-48976 | A05:2021 – Security Misconfiguration | Path traversal vulnerability in Commons FileUpload allowing arbitrary file write | **True Positive** | FileUpload implementation in dotCMS lacks comprehensive path normalization. `FileAssetFactory.java` and upload handlers don't consistently validate file paths against directory traversal. **Exploitability: HIGH** - Direct path to arbitrary file write if exploitable. |
| 8 | CVE-2024-47554 | A01:2021 – Broken Access Control | Path traversal vulnerability in Commons IO allowing access to files outside intended directory | **True Positive** | Commons IO used throughout filesystem operations. `APILocator.getFileAssetAPI()` and related file operations in `FileAssetAPIImpl.java` don't consistently sanitize paths. **Exploitability: MEDIUM-HIGH** - Could lead to sensitive file disclosure. |
| 9 | CVE-2024-25638 | A06:2021 – Vulnerable and Outdated Components | DNS cache poisoning vulnerability in dnsjava library | **False Positive** | While dnsjava is a dependency, dotCMS primarily uses it indirectly through Elasticsearch client. Direct DNS resolution by application code is minimal, limiting attack surface. Core CMS functionality doesn't expose DNS operations. **Exploitability: LOW** if present. |
| 10 | GHSA-crjg-w57m-rqqf | A06:2021 – Vulnerable and Outdated Components | DNS security vulnerability in dnsjava (KeyTrap attack) | **False Positive** | Same reasoning as CVE-2024-25638. DNSJava used as transitive dependency, not directly for DNSSEC validation in application logic. |
| 11 | GHSA-mmwx-rj87-vfgr | A06:2021 – Vulnerable and Outdated Components | DNSSEC validation bypass in dnsjava | **False Positive** | dotCMS doesn't implement DNSSEC validation in application layer. Library included transitively but vulnerability isn't exposed through CMS functionality. |
| 12 | CVE-2021-37136 | A04:2021 – Insecure Design | Netty codec vulnerability allowing request smuggling through improper HTTP/2 frame handling | **True Positive** | Netty is embedded in dotCMS for async operations and used by Elasticsearch client. Found in `dotCMS/core/dotCMS/src/main/java/com/dotmarketing/business/async/`. No custom validation of HTTP frames. **Exploitability: MEDIUM** - Requires specific network positioning and HTTP/2 support. |
| 13 | CVE-2021-37137 | A04:2021 – Insecure Design | Netty codec DoS through malformed HTTP/2 headers causing resource exhaustion | **True Positive** | Same Netty usage as CVE-2021-37136. No header size limits or validation beyond defaults in async handlers. **Exploitability: MEDIUM** - Can cause service degradation. |
| 14 | CVE-2012-6153 | A07:2021 – Identification and Authentication Failures | Apache HttpClient hostname verification vulnerability allowing MITM attacks | **True Positive** | HttpClient used in `com.dotmarketing.util.WebKeys` and external API integrations. | 15 | CVE-2021-40690 | A02:2021 – Cryptographic Failures | XML signature wrapping attack in Apache Santuario allowing signature bypass | **True Positive** | Santuario used for XML security in SAML implementation (`com.dotcms.saml.`). | 26 | CVE-2021-33813 | A05:2021 – Security Misconfiguration | XXE vulnerability in JDOM allowing external entity injection | **True Positive** | JDOM used for XML parsing in multiple locations including RSS feeds (`com.dotmarketing.portlets.rules.conditionlet.VisitorsCurrentUrlConditionlet`) without XXE protection configured. **Exploitability: HIGH** - XML processing from user input exists. |
| 27 | CVE-2024-21634 | A03:2021 – Injection | Amazon Ion Java stack overflow through deeply nested structures causing DoS | **True Positive** | Ion Java is dependency (likely through AWS SDK). **Summary:** 24 True Positives, 3 False Positives out of 27 HIGH/CRITICAL CVEs analyzed. Most critical concerns are:
1. **Immediate Risk**: Apache Tika PDF vulnerabilities (CVE-2025-66516, CVE-2025-54988) - RCE through file uploads
2. **High Risk**: Commons FileUpload path traversal (CVE-2025-48976) - arbitrary file operations
3. **High Risk**: JDOM XXE (CVE-2021-33813) - data exfiltration through XML processing
4. **Architectural Concern**: Multiple Bouncy Castle cryptographic weaknesses affecting system-wide security
